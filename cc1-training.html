第一部分：CC1 产品定位与核心能力

1.1 产品定位：CC1 不是"大而全"，是"小而全能"

  • 核心身份：4 合 1 通用清洁机器人（扫 / 吸 / 洗 / 拖），靠"一台机器覆盖多种场景"取胜，而不是靠"超大效率"。
  • 相对 MT1 的定位：MT1 打"大面积 + 干式 + 效率"，CC1 打"中小场景 + 多模式 + 能处理湿"。两者是互补关系，不是替代。
  • 最适合场景：
    • 酒店中 corridor（过道窄）、lobby 硬地面、游泳池周边 — 走廊窄、MT1 进不去时上 CC1。
    • 需要"干一台、湿一台"全包的中高端酒店。
    • 海底捞（Pudu 已部署约 2,000 台）类似的连锁餐饮 — 每天油污 + 水渍混合场景。
  • 反面场景：大仓库、大商超主通道、大宴会厅 — 这类地方用 MT1 更合适（CC1 清洁宽度窄、效率低，跑起来慢）。

1.3 核心差异：为什么不能"扫洗同时做"

客户最常问："既然是 4 合 1，为什么不能一次扫完 + 洗完？"本次培训给出的标准答案：

  • 扫地逻辑：靠滚刷旋转时的边缘线速度把垃圾"抛"进垃圾盒 —— 刷毛要硬、形状要能抛、密度要疏。
  • 洗地逻辑：靠刷毛压在地上刷洗，追求最大接触面积—— 刷毛要密、形状要贴地。
  • 物理冲突：两种刷毛形状 / 密度完全不同，一个刷子不可能同时做好两件事。实测做"前扫后洗"组合方案时：
    • 扫地性能下降一半以上（硬垃圾在滚刷间卡死）。
    • 洗地性能也下降（接触面积减少）。
    • 双滚刷洗地永远比单滚刷效果好 → 所以选择"扫时双滚刷扫、洗时双滚刷洗"，靠换件切换。
  • 销售话术："组合它是一个很好的宣传卖点，但做出来两头都差。我们选择让每个模式做到最好，换件只要一分钟。"

1.4 湿 / 液体处理能力

  • CC1 能处理湿垃圾 / 洒水— 这是 CC1 相对 MT1 的核心差异之一。
  • 典型场景：酒店走廊上红酒杯 / 玻璃水杯被碰洒 —— CC1 的吸水阀 + 污水箱可处理，MT1 则必须避障绕开。
  • 话术应用：酒店客户最怕"遇到湿就得再叫一个人" —— CC1 解决的正是这种随机性。
  • 注意：机器在 scrub 模式下会先放下刷子 + 等风机转起来，再开始移动（逻辑和 sweep 不同）。

第二部分：三种工作模式与组件切换

2.1 三种主要模式

模式底盘配件组合典型场景
Sweep（扫地） 双扫地滚刷 + 边刷 + 干式 filter 硬地面碎屑、粉尘、毛发；酒店大堂、corridor 日常清扫
Scrub（洗地） 洗地刷（密毛）+ 吸水阀 + 湿式 filter；无边刷 餐厅油污、酒店地面湿渍、pool 周边；最耗水最大吸力
Vacuum（吸尘） 吸尘爬（标配大件 / 选配小件） 地毯深吸、酒店走廊地毯、会议室地毯区域

2.2 吸尘爬（Vacuum Head）两种配件

  • 标配大吸尘爬（全宽）：覆盖面更广、通用性强 —— 日常出厂默认件。
  • 选配小吸尘爬（切了一半）：吸力更集中更强，但场景更窄 —— 用于地毯深层灰尘场景，效果最好。
  • 选件策略：
    • 酒店客户 → 建议用选配小吸尘爬（地毯吸得干净，客户直观看到效果）。
    • 大面积硬地面吸尘 → 用标配大件（效率 ≥ 效果）。
  • 话术要点："小的不是标配，是可选件 —— 它牺牲了覆盖宽度，换来了吸力密度，专门给地毯场景用。"

2.3 模式切换的标准流程（现场必背）

⚠ 这是 CC1 演示最容易翻车的环节，团队必须亲手反复练习。

从 Sweep 切到 Scrub 的完整步骤

  • ① 在 APP 里切 Task：新建或切换到 scrubbing task；系统会自动弹窗提示"请更换以下组件"。
  • ② 按提示换件（系统会一步步列出）：
    • 拆掉边刷（scrubbing 不用边刷）— 按住中间撸一下就下来。
    • 拆掉两个扫地滚刷，换成洗地刷。
    • 加装吸水阀（squeegee）— 位置在后部。
    • 换湿式 filter（干 filter 碰水会报废）。
  • ③ 确认：在 APP 里点"我已换好"；机器做自检，在位检测传感器确认所有组件就位。
  • ④ 启动任务。

刷子 / 组件安装要诀：怼到底、锁到位

  • 每个刷子底座是梅花轴 + 导向槽结构 —— 装的时候要对准，往下压到听到"咔哒"声。
  • 装不上的 90% 原因是"没怼到底"，不是坏件。发现扣不上时，先再往下压一次，再试。
  • 滚轮往上拨 → 锁住；整个装好后往下压不动 = 装到位了。
  • CC1 有"弹性可往下压"的设计，装到位后还能被压一小段，这是正常的（表示没卡死）。

2.4 关键保护：换不对组件机器不会跑

  • Filter 不换：机器不启动（干 filter 遇水直接报废）。
  • 刷子不换：可能启动但效果极差 + 刷头磨损异常。
  • 吸水阀没装：scrub 模式启动时报错。
  • 价值点：这套"在位检测 + APP 引导"保护的是客户 —— 就算前台员工临时换班、没培训过，也不会把机器用坏。

3.3 标准版 vs Pro 版 怎么选

  • 标准版（$22k）定位："雇合同工，不管干不干净，反正全扫一遍" —— 没有 AI 识别，任务按预定路径走完所有区域。
  • Pro 版（$24k）定位：多了 AI 识别 + Spot Cleaning（巡检模式）—— 哪脏扫哪，效率更高；AI 可识别酱油等液体脏污，自动降速、加水、加大吸力。
  • 销售判断：
    • 客户只看"干净标准一致" → 标准版就够。
    • 客户谈"夜间高效巡检 / 按需清洁 / 能看 AI 演示" → 推 Pro，$2k 差价很好打。
    • 客户要完整自动化闭环（免人工加水倒水） → 必推整套 $25k 含基站。
  • 基站卖点：自动加清水、自动排污水、自动充电 —— 酒店尤其买账（夜班人手更少，全自动更香）。

3.4 常见客户关切（CC1 特有）

  • "扫完湿垃圾咋办？" → CC1 先检测液体，可切 scrub 模式处理；或配套 MT1 做干净场景，CC1 专门处理湿区域。
  • "你家机器人换刷子太麻烦了。" → 现场演示一次：切 task → APP 提示 → 1 分钟换好。员工只要跟着 APP 走，不需要专门培训。
  • "为什么不做个家用款？" → Pudu 的赛道是商用耐操（Durability），家用市场价格战激烈，不是公司方向。告诉客户"我们 80% 收入来自海外商用场景，就是为了耐操"。
  • "能不能扫 + 洗一起做？" → 用 1.3 节的话术 —— 物理上不可能两头都好，换件 1 分钟解决。
  • "需要天花板贴 Marker 码，客户不乐意贴。" → Marker 提供全局唯一坐标，每 10 m 校验一次位置；对称结构场景特别需要。但我们也支持天花板自然特征定位（不贴码），只是大图 / 对称场景精度略差。跟客户解释"贴码 = 更稳，不贴 = 也能跑"。

第四部分：现场部署与调试

4.1 建图流程（与 MT1 基本一致）

  • ① 建图：把机器人放在 Home Point，APP 点 Start Mapping，手动推车沿边界走一圈；按主干道推即可，不必把每个死角都推到。
  • ② 地图清理：培训讲师建议的顺序 — 先排除安全风险（虚拟墙、禁区），再擦噪点；很多人顺序反过来，容易把玻璃 / 花瓶这类障碍物当成噪点擦掉。
  • ③ 虚拟墙 / 禁区：玻璃、镜面、楼梯口、花瓶、B 站（员工休息区 / backstage）这些雷达看不到或不希望进入的区域一定要画出来。
  • ④ 划定清洁区域：按 Sweep / Scrub / Vacuum 不同模式分别划区（同一块地允许多个模式的 task 共存）。
  • ⑤ 排程：设置每日 / 每周自动执行 + 勿扰时段。

4.2 定位标志物（Marker / 磁条 / 感光棒）

  • 天花板 Marker 码：
    • 作用：提供全局唯一坐标；每 10 m 自动校验位置。
    • 客户隐私：Pudu 对图像做隐私处理（不保留面部 / 人员识别），客户不用担心被监控。
    • 对称建筑（方形 lobby、矩形仓库）必须有绝对坐标点，Marker 码是最稳的方式。
  • 磁条（地面）：用于扶梯口 / 楼梯口物理阻挡 —— 但需要客户同意地面贴磁条，而且部署完贴完还要在 APP 里重新设置一次。
  • 感光棒 / 伊克尔加（一次性激光围栏）：部分客户询问的"放一个设备就不让机器进"方案；目前 Pudu 方案主要靠虚拟墙，硬件围栏属于极少数客户定制场景。

4.3 跨机型地图共用

  • 同一雷达高度的送餐机器人之间：可共用地图（不用重建图）。
  • 清洁机器人之间（CC1、MT1、CG one）：不能共用，必须各自重新建一次图 —— 因为雷达高度 / 清洁路径算法不同。
  • 3D 激光雷达机型（MT1 Pro 等）：也支持自己重新扫一遍地图，会自动校正。
  • 客户问"我有设备推过一次了，另一台要再推吗？"答：只要是清洁机型、雷达高度不同，就要再推一遍。

4.4 常见现场问题排查顺序

发生异常的标准排查顺序（口诀）：先看盘 → 看 hopper → 看刷子装没装到位 → 看 visual 里的虚拟墙和传感器显示。
  • ① 没吸上垃圾 / 漏垃圾：① 看底盘是否放下；② 看 hopper 有没有满；③ 看刷子有没有装到位（梅花轴、锁扣）。
  • ② 机器卡住 / 打转：打开 APP 的 visual 视图，看虚拟墙、看传感器识别的障碍点（橙色点 = 雷达 + RGB-D 实时障碍，蓝线 = 墙）。
  • ③ 切换中文界面：培训讲师提醒中文菜单里不显示造成垃圾的时间线；需要时切英文界面看原始信息。
  • ④ 机器人在某处反复徘徊：经 visual 确认是误识别后，在该区域设 Sensor Off / 传感器屏蔽区。

第五部分：演示策略与明日现场指南

5.1 演示重点：突出 CC1 的"能处理湿"和"能换模式"

  • 演示一：吸尘效果
    • 带自备粉末（面粉 / 爽身粉 / 咖啡渣），选客户的地毯区域撒一溜（不要一大片）。
    • 放慢速度（speed level 调低），效果最好 —— 慢 = 吸得更干净。
    • 清洁后打开 hopper / 污水箱直接给客户看。
  • 演示二：液体处理
    • 准备一杯水 / 模拟红酒（食用色素兑水），倒在硬地面小范围。
    • 切 scrub 任务 → 现场演示"CC1 遇到液体不避障，直接洗 + 吸干"。
    • 话术："酒店最怕客人把杯子碰洒，CC1 帮你少叫一次夜班清洁。"
  • 演示三：组件切换
    • 现场演示 1 分钟换件流程：拆边刷 → 换洗地刷 → 装吸水阀 → 换 filter → APP 确认启动。
    • 强调：任何员工看 APP 提示就能换，不需要专业培训。

5.2 明日现场准备清单

  • 主机：CC1 本体 + 充电桩（Station）如有则一起带。
  • 耗材 / 组件（一定要两套都带）：扫地滚刷 × 2、洗地刷 × 2、边刷 × 1、吸水阀（标配 + 备用）、干式 filter、湿式 filter、标配吸尘爬、选配小吸尘爬。
  • 演示道具：面粉 / 爽身粉 / 咖啡渣（撒一溜）、纸片碎屑、小瓶水（模拟液体泼洒）。
  • 其他：清洁工作证 / 胸牌、干擦镜布（擦传感器）、手机灯 / 手电（方便看轴位）。
  • 运输方式：抬起底盘锁定万向轮 → 像拖行李箱一样拖动；不要拆分装运（电池包 / 污水箱容易丢）。

5.3 演示节奏建议（针对酒店客户）

  • ① 先打安全牌：演示避障 —— 让机器人朝员工走过来，50 cm 内减速绕行。
  • ② 再打效果牌：在酒店 lobby 地毯撒粉末，吸完打开污水箱 / hopper 展示。
  • ③ 然后打差异化牌：演示液体处理 —— 这是 CC1 相对 MT1 / 竞品的核心差异。
  • ④ 最后打管理牌：展示 APP 里的 coverage 报告、AI 识别截图、远程监控界面。
  • ⑤ 收尾：算 ROI —— 酒店清洁工月薪 $2-3k，CC1 机身 $22-25k，替换 1 名全职可 1 年内回本；外加湿洒水、夜班不加人这些隐性成本节省。

5.4 明天到场的 Dos & Don'ts

  • Do：先把客户带到 CC1 的优势场景（窄 corridor、lobby、湿区）；演示前口头确认"吸尘头 / 刷子已放下"。
  • Do：模式切换慢一点、解释每一步为什么要换（建立专业感）。
  • Do：如果客户现场真的很干净，主动撒粉 / 洒水；在很干净的地上演示没效果 = 演砸。
  • Don't：不要让客户带你去大仓库 / 大停车场（那是 MT1 的场景，CC1 效率跑不过）。
  • Don't：不要让客户问"能不能扫洗同时"— 主动抢在前面解释，避免被牵着走。
  • Don't：不要只演标准版 —— Pro 版 AI 识别是最直观的"科技感"演示点，$2k 差价很容易升级。

第六部分：总结与后续行动

  • 产品总结：CC1 的核心价值是"一台机器覆盖酒店 / 餐饮 / 小场景的扫吸洗拖全流程"。它不追求 MT1 的大效率，追求的是"全能 + 能处理湿 + 窄过道也能进"。
  • 培训重点：
    • 模式切换流程（必反复练习到 1 分钟内完成）。
